# Модуль II — примеры (Python-ноутбуки)

Учебные примеры к темам 4–7. Сквозной пример — ассистент интернет-магазина «ШопБот». Каждый ноутбук самодостаточен, снабжён подробными пояснениями и запускается сверху вниз.

## Ноутбуки

| Файл | Тема | Содержание |
|------|------|-----------|
| `01_tools_tool_calling.ipynb` | 4. Инструменты | Инструмент как типизированный контракт; ручной stop-and-parse; структурированные ошибки, идемпотентность, компенсация; схема из Pydantic; цикл агента |
| `02_mcp_protocol.ipynb` | 5. MCP | MCP-сервер магазина (`FastMCP`); клиент, discovery, вызов; мост MCP→LLM; замена сервера без правки агента |
| `03_memory_and_rag.ipynb` | 6. Память и RAG | Кратко-/долговременная/контекстная память; entity memory; эмбеддинги + BM25; двухстадийный поиск; переписывание запроса; HyDE; RAG-ответ |
| `04_guardrails.ipynb` | 7. Защитный контур | Модель угроз; маскирование PII; детектор инъекций (regex + ML на реальном датасете); релевантность; валидация действий, allowlist, human-in-the-loop; инспекция вывода инструментов; метрика attack success rate |
| `05_langchain_langgraph.ipynb` | 4–6 *(на фреймворке)* | Тот же цикл «модель ⇄ инструменты» на **LangChain/LangGraph**: инструменты через `@tool`, готовый агент `create_agent`, кастомный граф (`StateGraph`, `ToolNode`, `tools_condition`), память через checkpointer. Все используемые методы/декораторы/функции обеих библиотек подробно разобраны в тексте |

Сопутствующие файлы: `mcp_shop_server.py`, `mcp_shop_server_v2.py` — MCP-серверы для ноутбука 02.

## Установка зависимостей

Python 3.12. Зафиксированные версии пакетов — в файле `requirements.txt`:

```bash
pip install -r requirements.txt
```

## Запуск

Открыть ноутбук в Jupyter или VS Code и выполнить ячейки по порядку. Либо из командной строки:

```bash
jupyter nbconvert --to notebook --execute --inplace 01_tools_tool_calling.ipynb
```

## Доступ к модели

Ноутбуки обращаются к языковой модели и эмбеддингам через **OpenAI-совместимый API**.
Настройте параметры доступа в первой (конфигурационной) ячейке каждого ноутбука —
удобнее через переменные окружения:

- `LLM_BASE_URL` — адрес эндпоинта (переменная `BASE_URL` / аргумент `base_url`);
- `LLM_API_KEY` — ключ (переменная `API_KEY`).

Подойдёт любой OpenAI-совместимый провайдер (OpenAI, локальный сервер вроде Ollama/vLLM
и т. п.). Если нет VPN или хочется без лишних настроек **пробовать разные модели**
(в т. ч. открытые и дешёвые), можно использовать провайдера-агрегатор
**[polza.ai](https://polza.ai)** — он предоставляет OpenAI-совместимый доступ к множеству
моделей из РФ; тогда `LLM_BASE_URL = https://polza.ai/api/v1`.

### Требования к моделям (можно экспериментировать с более лёгкими и дешёвыми)

Примеры не привязаны к конкретной модели — подставляйте любую, удовлетворяющую требованиям:

- **Чат-модель** с OpenAI-совместимым `chat/completions`. Референс в примерах —
  `qwen/qwen3.7-max`, но подойдёт и более лёгкая/дешёвая модель.
- **Поддержка вызова инструментов** (function/tool calling) — обязательна для тем 4 и 5.
- **Следование инструкциям формата** — модель должна уметь выдавать строгий ответ
  (например, `YES`/`NO` в классификаторе релевантности темы 7) и структурированный вывод.
- **Модель эмбеддингов** для темы 6 (RAG). Референс — `text-embedding-3-small`;
  подойдёт любая другая (например, `baai/bge-m3`), нужно лишь заменить `EMBED_MODEL`
  и пересчитать кэш `data/rag_embeddings.npy`.
- **Запас `max_tokens`** — для рассуждающих (reasoning) моделей часть токенов уходит
  на скрытые рассуждения, поэтому задавайте лимит с запасом.

Чтобы поменять модель, отредактируйте переменные `MODEL` / `EMBED_MODEL` в первой ячейке.
