## Правила выборки и проверки

| Параметр | Модуль I (аттестация) | Модуль II (лабораторная) |
|---|---|---|
| Число вопросов студенту | 2 | 2 (вопросы A и B) |
| Правило выборки | случайно, из **разных** тем | назначаются преподавателем в пары «атака/защита» |
| Форма проверки | гибридная: настроенная БЯМ + экспертная оценка | ИИ-агент-оценщик студента + эталонная проверка |
| Разрешение расхождений | регламент в [M1/kim-test.md](../../M1-Intelligent-agent/kim-test.md); решающий голос за преподавателем | регламент в [M2/rubric-01.md](../../M2-tools-knowledge-guardrails/rubric-01.md) |

Промпты оценивающих агентов вынесены в раздел [LLM-сервисы и промпты](../llm-prompts/README.md).

## Требования к добавлению

Для каждого банка укажите темы, количество заданий, уровни сложности, правила выборки, ключи и порядок обновления.

Дополнительно для данного курса: для вопросов открытого типа приведите опорные пункты эталонного ответа и рубрику — без них автоматическая проверка БЯМ невоспроизводима.
