# КИМ-3.0. Входная диагностика к модулю 3

**Модуль III · проводится до начала изучения тем 8–12**

> 🏷 **Тип задания:** тестовые задания закрытого типа · задания на установление соответствия · задания с кратким свободным ответом
> 🎯 **Когнитивный уровень (таксономия Блума):** **понимание и объяснение** (входной контроль)
> 📈 **Уровень компетенции (КРМ):** частично базовый — проверяются пререквизиты, а не материал модуля

## Назначение

Диагностический КИМ. Цель — **оценить текущие возможности студентов** перед изучением мультиагентных систем: усвоены ли инструменты, память и защитный контур из модуля 2, и есть ли база по графам, параллельным вычислениям и оценке производительности.

> ⚠️ **Результаты не входят в итоговую оценку по дисциплине.** Модуль 3 — самый нагруженный по предпосылкам: он одновременно опирается на модули 1–2 и на общеинженерную подготовку.

## Задания

### Блок A. Инструменты и протокол

**1.** *(выбор одного варианта)* Что представляет собой инструмент (tool) с точки зрения языковой модели?

- а) участок кода, который модель выполняет самостоятельно
- б) описанный контракт: имя, назначение и типы аргументов, который модель может запросить к вызову
- в) дополнительный набор весов модели
- г) способ увеличить контекстное окно

**2.** *(выбор одного варианта)* Кто фактически исполняет функцию, когда модель «вызывает инструмент»?

- а) сама языковая модель внутри себя
- б) программа-агент на стороне разработчика
- в) провайдер API автоматически
- г) операционная система

**3.** *(краткий свободный ответ)* Что даёт протокол MCP по сравнению с жёстко «зашитой» интеграцией инструментов в код агента?

### Блок B. Память и RAG

**4.** *(установление соответствия)* Сопоставьте вид памяти агента и его назначение:

| Вид | | Назначение |
|---|---|---|
| 1) кратковременная | | А) факты о сущностях, накапливаемые между сессиями |
| 2) долговременная | | Б) история текущего диалога |
| 3) контекстная | | В) данные, подставляемые в промпт под конкретный запрос |

**5.** *(краткий свободный ответ)* Зачем в конвейере RAG нужен этап переранжирования, если поиск уже вернул подходящие фрагменты?

### Блок C. Графы и структуры исполнения

**6.** *(выбор одного варианта)* В графе исполнения агента вершина (node) — это:

- а) переменная состояния
- б) шаг обработки (например, вызов модели или исполнение инструмента)
- в) условие перехода
- г) сообщение пользователя

**7.** *(краткий свободный ответ)* Чем условное ребро в графе исполнения отличается от безусловного? Приведите пример условия из работы агента.

### Блок D. Параллельные и распределённые вычисления

**8.** *(выбор одного варианта)* Три независимые подзадачи выполняются по 2 секунды каждая. Каково минимальное время при полностью параллельном исполнении (накладными расходами пренебречь)?

- а) 6 секунд
- б) 2 секунды
- в) 3 секунды
- г) зависит только от числа ядер процессора

**9.** *(выбор одного варианта)* Несколько параллельных ветвей записывают результат в общую структуру данных. Что необходимо предусмотреть?

- а) ничего, порядок гарантирован автоматически
- б) корректное объединение результатов, иначе записи могут затереть друг друга
- в) увеличить объём оперативной памяти
- г) выполнять ветви строго последовательно — иных вариантов нет

**10.** *(краткий свободный ответ)* Задача разделена между несколькими независимыми исполнителями. Назовите одно преимущество и одну издержку такого разделения по сравнению с одним исполнителем.

### Блок E. Производительность и ресурсы

**11.** *(выбор одного варианта)* Время отклика системы: медиана 0,8 с, p95 — 6 с. Что это означает?

- а) система всегда отвечает за 0,8 с
- б) половина запросов быстрее 0,8 с, но 5 % запросов медленнее 6 с
- в) среднее время равно 3,4 с
- г) измерение выполнено некорректно

**12.** *(краткий свободный ответ)* Что такое квантование модели и зачем его применяют при запуске на устройстве с ограниченной памятью?

## Ключи и эталонные ответы

*(раздел для преподавателя)*

| № | Ответ | Комментарий к проверке |
|---|---|---|
| 1 | **б)** описанный контракт | ключевой результат темы 4 |
| 2 | **б)** программа-агент | частая ошибка: считают, что модель исполняет код сама |
| 3 | Единый протокол позволяет подключать и заменять серверы инструментов без правки агента: инструменты обнаруживаются динамически, интеграция пишется один раз | 2 б. — есть заменяемость/переиспользование |
| 4 | 1–Б, 2–А, 3–В | |
| 5 | Первичный поиск оптимизирован на полноту и скорость и выдаёт много «похожего»; переранжирование более точной моделью упорядочивает фрагменты по реальной релевантности, чтобы в ограниченный контекст попали лучшие | 2 б. — названы и избыточность выдачи, и ограниченность контекста |
| 6 | **б)** шаг обработки | |
| 7 | Безусловное ребро всегда ведёт в одну и ту же вершину; условное выбирает следующий шаг по функции-маршрутизатору. Пример: «модель запросила инструмент → исполнить инструмент, иначе → завершить» | 2 б. — есть различие и корректный пример |
| 8 | **б)** 2 секунды | |
| 9 | **б)** корректное объединение результатов | прямая подготовка к reducer'ам состояния в МАС |
| 10 | Преимущество: специализация и параллельность (быстрее, надёжнее на своём участке). Издержка: расходы на координацию, обмен сообщениями, рост стоимости и сложности отладки | 2 б. — названы и то, и другое |
| 11 | **б)** половина быстрее 0,8 с, 5 % медленнее 6 с | базовый навык для бенчмарка в [КИМ-3.1](kim-01-practical-work-cloud.md) |
| 12 | Снижение разрядности весов (например, до 8 или 4 бит). Модель занимает меньше памяти и быстрее считается ценой некоторой потери качества — это позволяет запустить её на устройстве с ограниченными ресурсами | 2 б. — есть и «меньше памяти», и «ценой качества» |
