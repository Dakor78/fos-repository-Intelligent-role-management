# Модуль III — примеры (Python-ноутбуки)

Ориентир по архитектурам ИИ-агентов к темам 8–12. Сквозной домен — поддержка интернет-магазина «ШопБот». Все ноутбуки используют **общий набор инструментов** (`mas_domain.py`), поэтому архитектуры сравнимы между собой. Каждый ноутбук самодостаточен и запускается сверху вниз.

## Ноутбуки (по одной архитектуре/теме)

| Файл | Тема | Архитектура / содержание |
|------|------|--------------------------|
| `01_react_single_agent.ipynb` | 8 | Одиночный агент, цикл TAO (ReAct), stop-and-parse, трасса; тест на задачах возрастающей сложности |
| `02_interaction_types.ipynb` | 9 | Типы взаимодействия: коллаборация (handoff), дебаты, конкуренция |
| `03_mas_architectures.ipynb` | 10 | Иерархическая МАС: планировщик (structured output) + специалисты с ограниченными инструментами + координатор + критик; роутер-звезда |
| `04_a2a_protocols.ipynb` | 11 | Контракт межагентных сообщений (A2A) на Pydantic; разграничение MCP (агент↔инструмент) и A2A (агент↔агент); мультимодальные роли |
| `05_benchmark_and_model_economics.ipynb` | 10, 12 | Бенчмарк архитектур (single / MAS / MAS+critic) по качеству/вызовам/токенам/времени; дифференцированный подбор моделей под роли |
| `06_langgraph_mas.ipynb` | 10 *(на фреймворке)* | Иерархическая МАС на **LangGraph**: специалисты через `create_agent` с ограниченными инструментами; граф-супервизор с параллельным fan-out и узлом-координатором (reducer состояния). Все используемые методы/декораторы/функции LangChain и LangGraph подробно разобраны в тексте |

Вспомогательные модули:
- `mas_domain.py` — mock-домен и инструменты + их OpenAI-схемы (общие для всех ноутбуков).
- `mas_agents.py` — эталонные реализации архитектур с учётом метрик (используется в ноутбуке 05).

## Установка

Python 3.12. Зафиксированные версии пакетов — в файле `requirements.txt`:

```bash
pip install -r requirements.txt
```

Отдельные датасеты не нужны — учебный домен реализован в памяти (`mas_domain.py`).

## Доступ к модели (через переменные окружения)

Ключ и адрес **не вшиты в код** — читаются из окружения при запуске:

```bash
export LLM_API_KEY=<ваш-ключ>
export LLM_BASE_URL=https://<ваш-openai-совместимый-эндпоинт>/v1
export LLM_MODEL=<имя-модели>            # напр. qwen/qwen3.7-max
# для ноутбука 05 (два уровня моделей):
export LLM_MODEL_STRONG=<мощная-модель>          # напр. qwen/qwen3.7-max
export LLM_MODEL_CHEAP=<лёгкая-модель>           # напр. openai/gpt-4.1-nano
```

Запуск одного ноутбука:

```bash
jupyter nbconvert --to notebook --execute --inplace 01_react_single_agent.ipynb
```

Или открыть в Jupyter/VS Code и выполнить ячейки по порядку (задав переменные окружения заранее).

Подойдёт любой OpenAI-совместимый провайдер; можно экспериментировать с более лёгкими и дешёвыми моделями (требования к моделям — как в практической работе модуля).
