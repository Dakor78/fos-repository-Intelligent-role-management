# 📝 Экзамен

> 🏠 [К обзору курса](../README.md) · 📚 Материал: [🧠 Модуль 1](../M1-Intelligent-agent/README.md) · [🛠️ Модуль 2](../M2-tools-knowledge-guardrails/README.md) · [🕸️ Модуль 3](../M3-multi-agent-systems/README.md) · [📊 Модуль 4](../M4-evaluation-and-operations/README.md)

> 🏷 **Тип задания:** задание с открытым (свободным) ответом · собеседование по материалу дисциплины и артефакту курсового проекта
> 🎯 **Когнитивный уровень (таксономия Блума):** **понимание и объяснение → анализ** (обоснование инженерных решений)
> 📈 **Уровень компетенции (КРМ):** базовый → средний

Промежуточная аттестация подтверждает освоение дисциплины в целом и не должна дублировать только один модуль.

## Назначение

Экзамен проверяет владение материалом **всех четырёх модулей** и способность обосновывать инженерные решения: выбор между алгоритмом, одиночным агентом и МАС; проектирование инструментов и защитного контура; выбор архитектуры координации; способы оценки качества. Проверяемые результаты обучения — РО-1…РО-8 (см. [РПД](../docs/rpd.md)).

**Допуск к экзамену:** аттестация по каждому из четырёх модулей (см. корневой [README](../README.md), раздел «Итоговая оценка»).

## Процедура проведения

| Параметр | Значение |
|---|---|
| Форма | устный экзамен по билетам + обсуждение артефакта курсового проекта |
| Состав билета | 2 вопроса из [банка](question-bank.md), **из разных модулей** (случайная выдача) |
| Подготовка | 40 минут, без конспектов и без доступа к БЯМ; разрешён чистый лист для плана ответа |
| Ответ | 15–20 минут: по 5–7 минут на вопрос + 3–5 минут обсуждение проекта |
| Максимум | 100 баллов |
| Положительная оценка | от 60 баллов **и** не менее 16 баллов за каждый вопрос билета |

**Обсуждение артефакта курсового проекта.** Экзаменатор задаёт 2–3 вопроса о решениях, принятых студентом в [курсовой работе](../Project/README.md) или, если она ещё не завершена, в практических работах модулей: почему выбрана такая ролевая структура, почему такие инструменты, как оценивалось качество. Проверяется способность **обосновать собственные** решения материалом курса, а не воспроизвести заготовленный текст.

## Критерии оценивания (100 баллов)

### Вопросы билета — по 40 баллов за вопрос

| № | Критерий | Баллы |
|---|----------|:-----:|
| 1 | **Фактическая корректность** — терминология и утверждения верны, ошибок по существу нет | 16 |
| 2 | **Полнота** — раскрыты опорные пункты ответа | 12 |
| 3 | **Обоснованность** — выводы аргументированы, видна причинно-следственная связь | 8 |
| 4 | **Изложение** — ответ структурирован, формулировки однозначны | 4 |
| | **Итого за вопрос** | **40** |

### Обсуждение артефакта проекта — 20 баллов

| № | Критерий | Баллы |
|---|----------|:-----:|
| 5 | **Обоснованность собственных решений** — студент объясняет, почему выбраны данная архитектура, роли, инструменты и способ оценки, и какие альтернативы были отвергнуты | 12 |
| 6 | **Связь с материалом курса** — обоснование опирается на понятия дисциплины (методологический принцип, контракты инструментов, метрики качества), а не на общие слова | 8 |
| | **Итого** | **20** |

**Штрафы:**
- −8 б. за вопрос — ответ подменяет объяснение пересказом определения без применения к вопросу;
- −16 б. за вопрос — грубая понятийная ошибка (например, отождествление агента с одиночным запросом к БЯМ);
- 0 за вопрос — ответ не по существу.

Перевод в пятибалльную шкалу — по общей таблице курса (корневой [README](../README.md), раздел «Итоговая оценка»).

## Порядок проверки

Ответы оцениваются по гибридной схеме, принятой в курсе: независимая оценка настроенной БЯМ (по опорным пунктам и рубрике) и экспертная оценка преподавателя; решающий голос — за человеком. Регламент — калибровка оценщика (κ ≥ 0,6), независимость оценок, пороги разрешения расхождений 10/20 баллов — единый для всей дисциплины и описан в [методических указаниях по оцениванию](../methodical-guidelines/teachers-assessment/README.md). Для устной формы допускается запись тезисов ответа с листа подготовки студента.

## Банк вопросов

[question-bank.md](question-bank.md) — 32 вопроса: по 8 на каждый модуль, с интегральными формулировками «объясните и обоснуйте». Банк открыт студентам заранее: проверяется не память на формулировки, а способность объяснять.

## Подготовка

- Пройти банки вопросов модулей: [КИМ-1.4](../M1-Intelligent-agent/kim-test.md) (32 вопроса), [КИМ-2.2](../M2-tools-knowledge-guardrails/kim-02-open-questions.md) (40 вопросов) — экзаменационные вопросы построены на тех же понятиях, но требуют связывать материал разных модулей.
- Повторить **собственные** работы: лучший способ подготовки к обсуждению артефакта — уметь объяснить каждое принятое в них решение.
- Демонстрационные ноутбуки модулей — быстрый способ освежить механику (цикл агента, MCP, RAG, архитектуры МАС, eval).
