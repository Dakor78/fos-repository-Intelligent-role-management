# 🧠 Модуль 1. Интеллектуальный агент и его вычислительное ядро

> 🏠 [К обзору курса](../README.md) · ◀️ **Предыдущие модули:** это первый модуль курса — предшествующих нет.
>
> 📋 **Что нужно знать заранее:** Python, основы машинного обучения, базовые навыки промпт-инженерии.

**Зачем этот модуль.** Прежде чем строить агентные системы, нужно ответить на два вопроса: *нужен ли здесь вообще ИИ-агент* и *что языковая модель принципиально не умеет*. Модуль отвечает на оба и завершается первым работающим агентом.

---

## 📚 Темы модуля

| № | Тема | Ключевые понятия |
|---|------|------------------|
| 1 | Введение. От промптинга к распределённым мультиагентным системам | агент = исполняемая среда (runtime) + объединяющая языковую модель + системную инструкцию + инструменты + память + механизмы защиты и планирования |
| 2 | Большая языковая модель как вычислительное ядро агента | представление БЯМ, промпты, токены; обучение; сильные и слабые стороны БЯМ; применимость |
| 3 | Базовый агент: исполняемая среда, цикл работы и ролевая системная инструкция | системная инструкция (системный промпт) как носитель роли агента: цель, ограничения, стилистические требования, политика принятия решений. Уровни сообщений диалога: system / user / assistant / tool |

|-----|-----|-----------|------|
| КИМ 1.1 | 🔬 Лабораторная работа | Аналитический обзор: выбор предметной области; описание сценария, в котором ролевое распределение функций между агентами является обоснованным; аргументация выбора по критерию «детерминированный алгоритм / одиночный агент / мультиагентная система» | [kim-lab-01.md](kim-lab-01.md) |
| КИМ 1.2 | 🔬 Лабораторная работа | Лабораторная работа по экспериментальному исследованию поведения БЯМ: сравнение локального инференса и обращения к внешнему API по критериям конфиденциальности данных, стоимости и времени отклика; эксперименты с токенизацией, температурой и стратегиями сэмплирования; воспроизведение и интерпретация типовых отказов модели | [kim-lab-02.md](kim-lab-02.md) |
| КИМ 1.3 | 🔬 Лабораторная работа | Реализация базового агента с явным описанием роли в системной инструкции; построение графа исполнения; вывод полной трассы шагов исполнения | [kim-lab-03.md](kim-lab-03.md) |
| КИМ 1.4 | 📝 Аттестация по разделу | Проверочная работа на знание алгоритмов и определений: 2 вопроса открытого типа, гибридная проверка (настроенная БЯМ + экспертная оценка) | [kim-test.md](kim-test.md) |

Каждая лабораторная работа содержит цели, условия выполнения, пошаговый ход работы, состав сдаваемых материалов, критерии оценивания на 100 баллов, типовые ошибки и связь с последующими модулями. Для КИМ 1.1 приведён банк из 10 вариантов предметных областей, для КИМ 1.4 — банк из 32 вопросов.

> 🔗 **Сквозная линия модуля.** Работы выполняются последовательно на одной предметной области: в КИМ 1.1 выбирается область и составляется упрощённое ТЗ, в КИМ 1.2 экспериментально подбираются параметры моделей для её ролей, в КИМ 1.3 одна из ролей реализуется как работающий агент.
>
> Выбирайте область, с которой готовы работать до конца курса, — она переиспользуется в модулях 2–4 и в курсовом проекте.
