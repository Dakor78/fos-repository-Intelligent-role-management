# Методические указания преподавателям по оцениванию

Документ определяет единый порядок оценивания по дисциплине: шкалы, работу с рубриками, применение языковых моделей при проверке, обратную связь и апелляции.

## Общие принципы

1. **Оценивается результат, а не объём.** Во всех практических работах курса баллы начисляются за обоснованность инженерных решений, а не за количество написанного кода или страниц отчёта.
2. **Критерии известны заранее.** Рубрика каждой работы опубликована в её КИМ и доступна студенту до начала выполнения. Оценивание по неопубликованным критериям не допускается.
3. **Базовый минимум раздела.** В ряде работ выделен ключевой критерий, без набора минимального балла по которому работа не аттестуется независимо от общей суммы. Это защищает от компенсации непонимания сути аккуратным оформлением.
4. **Решающий голос за преподавателем.** Языковая модель применяется как вспомогательный инструмент проверки; итоговое решение принимает человек.
5. **Воспроизводимость.** Работа, результаты которой невозможно воспроизвести (нет кода, не зафиксированы версии, ключи не вынесены в окружение), не может быть оценена выше порога аттестации.

## Шкалы и пороги

Каждая работа оценивается по 100-балльной шкале внутри своего КИМ. Балл модуля — взвешенное среднее оценок его КИМ, вклад модуля в итоговую оценку курса — `баллы модуля × 0,2` с арифметическим округлением (до 20 баллов за модуль):

| Модуль | Свёртка оценок КИМ |
|---|---|
| [Модуль 1](../../M1-Intelligent-agent/README.md) | КИМ-1.1 × 0,3 + КИМ-1.2 × 0,25 + КИМ-1.3 × 0,25 + КИМ-1.4 × 0,2 |
| [Модуль 2](../../M2-tools-knowledge-guardrails/README.md) | КИМ-2.1 × 1,0 |
| [Модуль 3](../../M3-multi-agent-systems/README.md) | КИМ-3.1 × 0,5 + КИМ-3.2 × 0,5 |
| [Модуль 4](../../M4-evaluation-and-operations/README.md) | КИМ-4.1 × 1,0 |

Входные диагностики (КИМ-N.0) в свёртке не участвуют. **Условие аттестации по модулю:** каждая входящая в свёртку работа сдана на не менее 60 баллов; компенсация несданной работы высокими баллами за другие не допускается. Полное описание системы итогового оценивания — в корневом [README](../../README.md), раздел «Итоговая оценка».

| 100-балльная оценка | Пятибалльная | Сокр. | Код |
|---|---|---|---|
| 0 – 59 | неудовлетворительно | неуд | 2 |
| 60 – 69 | удовлетворительно | уд | 3 |
| 70 – 89 | хорошо | хор | 4 |
| 90 – 100 | отлично | отл | 5 |

Аттестация по разделу — не менее 60 баллов **и** выполнение базового минимума раздела, если он определён в КИМ.

## Где находятся рубрики

| Модуль | Работа | Где критерии |
|---|---|---|
| I | КИМ-1.1 … КИМ-1.4 | внутри каждого файла КИМ, раздел «Критерии оценивания» |
| II | КИМ-2.1 (состязательная лабораторная) | отдельный файл [rubric-01.md](../../M2-tools-knowledge-guardrails/rubric-01.md) |
| III | КИМ-3.1, КИМ-3.2 | внутри каждого файла КИМ (по 100 баллов на часть) |
| IV | КИМ-4.1 | внутри файла КИМ |

Различие объясняется характером работ: рубрика модуля II вынесена отдельно, поскольку описывает состязательную процедуру с начислением за атаку и защиту.

## Применение языковых моделей при проверке

В курсе БЯМ используется для проверки открытых ответов (КИМ-1.4) и как объект разработки студентом (КИМ-2.1). Обязательный порядок:

1. **Калибровка до применения.** Агент-оценщик калибруется на выборке 15–20 ранее проверенных ответов; согласованность с экспертом измеряется каппой Коэна, приемлемый уровень — κ ≥ 0,6. Некалиброванный оценщик к работе не допускается.
2. **Независимость оценок.** Преподаватель выставляет оценку, **не видя** оценки модели, — иначе возникает эффект якорения.
3. **Обезличивание.** Ответы подаются модели без персональных данных студента.
4. **Детерминированность.** Оценивание выполняется при `temperature=0`; фиксируются модель, версия и промпт.
5. **Разрешение расхождений** — по регламенту в [M1/kim-test.md](../../M1-Intelligent-agent/kim-test.md):

| Расхождение | Действие |
|---|---|
| ≤ 10 баллов | итог — оценка эксперта |
| 11–20 баллов | эксперт перепроверяет, видя обоснование модели |
| > 20 баллов | привлекается второй эксперт, итог — среднее двух экспертных оценок |

Промпты оценивающих агентов и требования к ним — в разделе [LLM-сервисы и промпты](../../resources/llm-prompts/README.md).

## Типовые сложности при оценивании

| Ситуация | Рекомендация |
|---|---|
| Отчёт аккуратен, но выводы не опираются на собственные измерения | Снижать по профильному критерию: пересказ документации результатом эксперимента не является (штраф предусмотрен в КИМ-1.2) |
| Все операции отнесены студентом к МАС | Прямое нарушение методологического принципа курса; в КИМ-1.1 предусмотрен штраф −20 баллов |
| Работа воспроизводит пример из материалов курса на учебном домене | Работы выполняются на **своей** предметной области; совпадение с демонстрационным примером — основание вернуть на доработку |
| Агент отвечает на любой запрос, включая запрещённый ролью | Отсутствие отказного поведения — дефект, а не мелочь; в КИМ-1.3 предусмотрен штраф −15 баллов |
| Обнаружен ключ доступа в коде или отчёте | Штраф −20 баллов во всех работах, где предусмотрен; требовать отзыва ключа |
| Недетерминированность результатов между запусками | Норма для БЯМ. Требовать не одинаковых чисел, а корректной методики: повторы, медиана, разброс |

## Обратная связь

Обратная связь предоставляется по структуре рубрики: по каждому критерию — набранный балл и краткое обоснование. Рекомендуется указывать не только недостатки, но и конкретное действие для их устранения, поскольку результаты работ переиспользуются в последующих модулях (постановка задачи из модуля I — вход для модулей II–IV).

При использовании агента-оценщика его обоснование может передаваться студенту как дополнительный комментарий с явной пометкой, что оценка выставлена преподавателем.

## Апелляции

1. Апелляция подаётся в течение установленного срока после публикации оценки и содержит указание на конкретные критерии рубрики.
2. Апелляция по существу задания (несогласие с самой постановкой) не принимается — критерии публикуются до начала работы.
3. Рассмотрение проводится вторым преподавателем; при расхождении более 20 баллов итоговой считается средняя оценка.
4. Если расхождение вызвано ошибкой агента-оценщика, случай включается в калибровочную выборку для последующей корректировки промпта.

## Академическая добросовестность

Использование генеративных моделей студентами разрешено и является частью дисциплины. Оценивается способность **обосновать** полученный результат: на защите студент должен объяснить принятые решения и ответить на вопросы по своему коду. Неспособность объяснить сданный результат — основание для снижения оценки независимо от качества артефакта.
