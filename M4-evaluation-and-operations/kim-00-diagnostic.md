# КИМ-4.0. Входная диагностика к модулю 4

**Модуль IV · проводится до начала изучения тем 13–16**

> 🏷 **Тип задания:** тестовые задания закрытого типа · задания на установление соответствия · задания с кратким свободным ответом
> 🎯 **Когнитивный уровень (таксономия Блума):** **понимание и объяснение** (входной контроль)
> 📈 **Уровень компетенции (КРМ):** частично базовый — проверяются пререквизиты, а не материал модуля

## Назначение

Диагностический КИМ. Цель — **оценить текущие возможности студентов** перед изучением оценки качества и эксплуатации: есть ли работающая агентная система из модуля 3, понимание её трассы и базовые представления о тестировании и статистике согласия.

> ⚠️ **Результаты не входят в итоговую оценку по дисциплине.** Модуль 4 работает с системой, построенной в модуле 3: если её нет или её поведение непонятно студенту, оценивать будет нечего.

## Задания

### Блок A. Система и её трасса

**1.** *(выбор одного варианта)* Что должна содержать полная трасса исполнения агента, чтобы по ней можно было понять причину неверного ответа?

- а) только финальный ответ агента
- б) шаги с входами и выходами: обращения к модели, вызовы инструментов и их результаты
- в) исходный код агента
- г) список установленных библиотек

**2.** *(краткий свободный ответ)* Агент отказался выполнить запрос, потому что это запрещено его ролью. Является ли отказ ошибкой? Обоснуйте.

**3.** *(выбор одного варианта)* Роль агента-критика в мультиагентной системе:

- а) ускорять работу остальных агентов
- б) независимо проверять результат работы других агентов
- в) распределять задачи между исполнителями
- г) хранить историю диалога

### Блок B. Основы тестирования

**4.** *(выбор одного варианта)* Что такое регрессионное тестирование?

- а) проверка того, что новые изменения не сломали ранее работавшую функциональность
- б) тестирование только новых возможностей
- в) нагрузочное тестирование
- г) построение регрессионной модели по данным

**5.** *(установление соответствия)* Сопоставьте вид тестового случая и его назначение:

| Вид | | Назначение |
|---|---|---|
| 1) позитивный | | А) проверка поведения на некорректных входных данных |
| 2) негативный | | Б) проверка штатного сценария |
| 3) граничный | | В) проверка на предельных значениях диапазона |

**6.** *(краткий свободный ответ)* Почему набор тестов, состоящий только из позитивных сценариев, даёт завышенное представление о качестве системы?

### Блок C. Недетерминированность и статистика

**7.** *(выбор одного варианта)* Система с недетерминированным поведением прошла тест с первого раза. Что из этого следует?

- а) система работает корректно
- б) тест пройден однократно; для вывода о надёжности нужны повторные прогоны
- в) тест составлен неверно
- г) можно исключить этот тест из набора

**8.** *(краткий свободный ответ)* Задача выполнялась 4 раза, успешно — 2 раза. Как бы вы охарактеризовали такую систему заказчику, которому важна предсказуемость?

**9.** *(выбор одного варианта)* Изоляция тестовых прогонов означает, что:

- а) тесты выполняются на отдельном компьютере
- б) каждый прогон начинается с одинакового исходного состояния и не зависит от результатов предыдущих
- в) тесты запускаются строго по одному
- г) результаты тестов не публикуются

### Блок D. Метрики и согласованность оценок

**10.** *(краткий свободный ответ)* Два эксперта независимо оценили одни и те же 100 работ и совпали в 80 % случаев. Достаточно ли доли совпадений, чтобы утверждать, что оценки согласованы? Что здесь не учтено?

**11.** *(выбор одного варианта)* Для проверки того, что итог работы системы верен, надёжнее сравнивать:

- а) текст ответа с эталонным текстом дословно
- б) итоговое состояние системы (что фактически изменилось) с ожидаемым
- в) длину ответа
- г) время выполнения

### Блок E. Эксплуатация и стоимость

**12.** *(краткий свободный ответ)* Мультиагентная система дала на 5 % более высокое качество, чем одиночный агент, но обошлась в 4 раза дороже и работает вдвое дольше. Какие сведения нужны, чтобы решить, стоит ли внедрять её в эксплуатацию?

## Ключи и эталонные ответы

*(раздел для преподавателя)*

| № | Ответ | Комментарий к проверке |
|---|---|---|
| 1 | **б)** шаги с входами и выходами | результат модуля 1 |
| 2 | Нет, не является: корректный отказ — правильный результат работы. Роль задаёт границы, и их соблюдение так же важно, как выполнение разрешённых действий | 2 б. — есть прямой ответ «не ошибка» и обоснование через роль |
| 3 | **б)** независимо проверять результат | |
| 4 | **а)** проверка, что изменения не сломали работавшее | |
| 5 | 1–Б, 2–А, 3–В | |
| 6 | Позитивные сценарии проверяют только штатное поведение; система, которая никогда не отказывает и не обрабатывает ошибки, пройдёт их все, оставаясь непригодной. Дефекты проявляются на некорректных и граничных входах | 2 б. — есть мысль о непокрытых классах ситуаций |
| 7 | **б)** нужны повторные прогоны | ключевая идея pass^k |
| 8 | Система ненадёжна: успех в половине случаев означает, что результат нельзя предсказать заранее. Для заказчика важна не разовая удача, а воспроизводимость; в текущем виде требуется ручная проверка каждого результата | 2 б. — вывод о ненадёжности и упоминание предсказуемости/повторяемости |
| 9 | **б)** одинаковое исходное состояние, независимость прогонов | |
| 10 | Недостаточно: часть совпадений возникает случайно, особенно при неравномерном распределении оценок. Нужна мера, учитывающая случайное согласие (каппа Коэна) | 2 б. — названа случайная составляющая согласия; 1 б. — «нужна дополнительная метрика» без объяснения |
| 11 | **б)** итоговое состояние системы | центральный принцип модуля |
| 12 | Нужны: требования к качеству (окупают ли 5 % затраты), допустимое время отклика, бюджет и объём нагрузки, цена ошибки в предметной области, а также надёжность выигрыша — устойчив ли он при повторных прогонах или лежит в пределах разброса | 2 б. — названы минимум два фактора, включая цену ошибки или устойчивость выигрыша |
