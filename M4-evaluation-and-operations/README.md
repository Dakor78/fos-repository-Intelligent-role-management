# 📊 Модуль 4. Оценка качества и промышленная эксплуатация

> 🏠 [К обзору курса](../README.md) · ◀️ **Предыдущие модули:** [🧠 Модуль 1](../M1-Intelligent-agent/README.md) · [🛠️ Модуль 2](../M2-tools-knowledge-guardrails/README.md) · [🕸️ Модуль 3](../M3-multi-agent-systems/README.md)
>
> 📋 **Что нужно знать заранее:** трасса шагов агента (модуль 1); негативные сценарии и защитный контур (модуль 2); построенная мультиагентная система, которая здесь становится объектом оценки (модуль 3).

Директория содержит контрольно-измерительные материалы (КИМ) модуля 4 и демонстрационные материалы. Модуль охватывает оценку качества (evaluation) агентных систем как самостоятельную инженерную дисциплину и их промышленную эксплуатацию.

**Зачем этот модуль.** «Работает на моих примерах» — не результат. Заключительный модуль превращает оценку качества в инженерную дисциплину: воспроизводимые прогоны, измеримые критерии, калиброванный судья и контроль деградаций при выходе в эксплуатацию.

---

## 📚 Темы модуля

| № | Тема | Ключевые понятия |
|---|------|------------------|
| 13 | Постановка задачи и набор тестовых задач | корзина задач (evaluation suite), полная структура задачи, теги, негативные/граничные/двунаправленные случаи |
| 14 | Инфраструктура прогонов и критерии | eval harness (изоляция, параллельность, трассировка), Iron User, критерии результата/траектории, pass@k / pass^k |
| 15 | Оценивание, LLM-судья и эталонные наборы | code-based + LLM-судья, калибровка по каппе Коэна, публичные бенчмарки |
| 16 | Эксплуатация распределённых агентных систем | наблюдаемость, регрессия, дифференцированный подбор моделей, метрики время/стоимость/ошибки, пороги качества |
